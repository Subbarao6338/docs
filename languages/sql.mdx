SQL (Structured Query Language) is a standard language for designing, creating, and managing relational databases.

### SQL Joins Comparison

Understanding how to combine data from multiple tables is crucial.

| Join Type | Description | Result |
| :--- | :--- | :--- |
| `INNER JOIN` | Matches in both tables | Returns rows with matching values in both tables |
| `LEFT JOIN` | All from left, matches from right | Returns all rows from left, and matched rows from right |
| `RIGHT JOIN` | All from right, matches from left | Returns all rows from right, and matched rows from left |
| `FULL JOIN` | All rows from both | Returns rows when there is a match in one of the tables |

### Advanced Queries ⚡

#### Common Table Expressions (CTEs)
CTEs provide a way to write more readable and modular SQL by defining temporary result sets.

```sql
WITH regional_sales AS (
    SELECT region, SUM(amount) AS total_sales
    FROM orders
    GROUP BY region
)
SELECT region, total_sales
FROM regional_sales
WHERE total_sales > (SELECT AVG(total_sales) FROM regional_sales);
```

#### Window Functions
Window functions perform calculations across a set of table rows that are somehow related to the current row.

```sql
SELECT
    employee_id,
    department,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) as salary_rank,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as global_rank
FROM employees;
```

### Syntax Cheat Sheet

#### Data Query Language (DQL)
```sql
SELECT column1, column2
FROM table_name
WHERE condition
ORDER BY column1 ASC|DESC;
```

#### Data Manipulation Language (DML)
```sql
INSERT INTO table_name (col1, col2) VALUES (val1, val2);
UPDATE table_name SET col1 = val1 WHERE condition;
DELETE FROM table_name WHERE condition;
```

### Optimization Tips 🚀

Writing queries that run fast is essential when working with large datasets.

#### Indexing Strategies
- **Primary Keys**: Ensure every table has a primary key.
- **Composite Indexes**: Use for queries that filter on multiple columns.
- **Explain Plan**: Always analyze your query's execution plan.
  ```sql
  EXPLAIN ANALYZE SELECT * FROM orders WHERE user_id = 123;
  ```

<Check>
  **Use Indexes**: Create indexes on columns that are frequently used in `WHERE` clauses to speed up lookups.
</Check>

<Warning>
  **Avoid SELECT \***: Specify exactly which columns you need. This reduces the amount of data transferred and can improve performance.
</Warning>

<Tip>
  **Limit Your Results**: Use `LIMIT` (PostgreSQL/MySQL) or `TOP` (SQL Server) during development to avoid accidentally fetching millions of rows.
</Tip>
